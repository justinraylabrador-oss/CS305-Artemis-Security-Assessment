# CS305-Artemis-Security-Assessment
Artifact

This repository contains my Artemis Financial Practices for Secure Software Report, which demonstrates my ability to analyze vulnerabilities and apply secure coding practices to improve software security.

Client Summary

The client in this project was Artemis Financial, a financial services company that develops software to manage financial data and transactions. The company wanted to improve the security of its application by identifying vulnerabilities and implementing secure coding practices. The goal was to ensure that sensitive financial information could be transmitted securely and protected from potential cyber threats.

Security Vulnerability Assessment

While analyzing the software, I identified potential vulnerabilities in dependencies and reviewed the project’s security risks. Coding securely is important because it helps prevent data breaches and protects sensitive information from attackers. Software security adds value to a company by protecting customer trust, preventing financial loss, and ensuring systems remain reliable and safe.

Challenges and Helpful Parts

One of the more challenging aspects of the vulnerability assessment was understanding how dependency scanning tools identify security risks. However, learning how to interpret these results was helpful because it showed how automated tools can quickly detect vulnerabilities in software components.

Increasing Layers of Security

I increased layers of security by implementing secure communication methods and reviewing project dependencies for known vulnerabilities. In the future, I would use vulnerability scanning tools, static code analysis tools, and secure development practices to assess risks and determine the best mitigation techniques.

Ensuring Functional and Secure Code

To make sure the application remained functional and secure, I tested the application after making changes and ran vulnerability scans again to confirm that no new security issues were introduced during refactoring.

Tools and Resources Used

During this project, I used dependency-check tools, secure coding practices, and vulnerability analysis techniques. These resources are useful for identifying security risks early in the development process and maintaining secure software systems.

Portfolio Value

This project demonstrates my ability to analyze software vulnerabilities, apply secure coding practices, and improve application security. I could show this artifact to future employers as an example of my understanding of software security, vulnerability mitigation, and secure development processes.
